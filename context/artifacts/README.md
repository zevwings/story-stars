# 动态器物目录说明

> 本页说明 `context/artifacts/` 的目录用法。动态器物登记入口见 [_index.md](./_index.md)；静态设定仍以 `worldbuilding/` 或正式器物条目为准。

## 用途

- 记录器物当前归属、损耗、封存、解锁、权限、灵力状态和剧情承接。
- 辅助判断正文中某件器物现在能不能使用、由谁持有、是否已经暴露。
- 保留已经定稿或作者确认的动态变化，不补写尚未进入正文的器物事实。

## 使用规则

- 查器物当前状态时，先读 [世界观索引](../../worldbuilding/_index.md) 对应静态设定或正式器物条目，再读本目录对应动态文件。
- 动态状态缺失时，不得用剧情套路补齐。
- 不把静态器物设定重复写成长篇说明。
- 不因为本目录存在就默认某件器物已经进入正文。
- 只有定稿、补录摘要或上下文整理任务确认器物状态变化后，才更新本目录。

## 动态状态文件格式

每件需要追踪的器物在本目录下有独立文件 `{器物名}.md`。新建时复制 [_template.md](./_template.md)，再按正式章节或作者确认内容填写。

## 变更日志维护规则

1. 每章写完后，检查本章是否引起了器物状态变化，包括归属、损耗、封印、权限、暴露程度、灵力状态和使用限制。
2. 如果有变化，更新对应器物字段，并在“状态变更记录”中追加一行，格式为 `- 第{N}章：{变化描述}`。
3. 如果无变化，跳过。
4. 仅追踪关键器物、长期随身物、重要信物、会牵动线索或会影响战斗、修炼、身份判断的器物，不追踪一次性杂物。
5. **压缩规则**：每件器物保留最近 10 条详细变更记录。超过 10 条时，将最早的记录压缩为摘要写入对应字段，删除已合并的旧记录行。

## 与写前读取的关系

写作每章前，只读取当前任务明确涉及的器物动态文件。不是全量读取，只筛选 `outline.md`、章节承接、人物当前状态或作者当次任务中提到的器物。
