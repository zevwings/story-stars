# 人物模板说明

> 本目录只负责人物静态模板分流，不横向展开具体人物正文或动态状态。模板存在不代表人物已经进入正典。本模板按本项目都市星辰契约世界观维护；不得把旧项目的修炼、法器、化形、双魂等体系词汇带回卡内。

## 选用说明

- 人物统一使用三个必备页：`主卡.md`、`形象.md`、`衣装.md`；确认需要角色专属契约档案时，按需增加 `契约.md`。
- 先选目录层级，再复制必备模板；主次与登场状态由 `major` / `minor` / `preentry` 目录和对应索引承担。
- `major` 适合主角、核心配角、关键对手、长期回归角色。
- `minor` 适合阶段性配角、一次性推动角色、信息承载角色。
- `preentry` 适合尚未正式登场、但已经完成设定的人物。
- 人物编号沿用 `characters/_index.md` 的登记口径（如 `M-01`、`F-01`），编号不随目录层级迁移而改变。

## 文件职责

- `主卡.md` 写“这个人是谁、怎么行动、和谁有关、边界是什么”，只保留星辰契约摘要，不展开能力机制。
- `契约.md` 写角色实际持有的星辰、权柄、具体能力、个人限制、登记状态与异变表现，不重复人物性格、职业和关系正文。
- `形象.md` 写“这个人长什么样、身体与气质如何被正文调用”。
- `衣装.md` 写“这个人穿什么、怎么妆造、不同场景如何调用”。
- 动态状态（当前处境、关系推进、出场记录）写入 `context/characters/人物名.md`，不要塞回静态卡。
- `major` 可以写厚，但复杂能力细则仍应进入 `契约.md`，不以超长段落堆在主卡。
- `minor` 保持轻量，重点保证功能清楚、形象完整、关系可调用。

## 契约档案创建条件

- **非契约者**：不创建 `契约.md`，主卡星辰契约摘要写“不适用”。
- **契约状态未确认**：不创建正式 `契约.md`；能力候选与缺口转入施工层待确认记录，主卡不设占位字段。
- **已确认是契约者，但只定权柄方向**：可以创建轻量 `契约.md`，只写已确认方向与边界；其余内容登记于施工层。
- **权柄、具体能力或星辰异变已经确认**：创建完整 `契约.md`，并由主卡摘要链接过去。
- 不为追求模板完整度保留空小节；没有复合能力、异变细则或暴露管理时，删除对应章节。

## 星辰契约真源边界

- `worldbuilding/systems/星辰契约.md`、`星辰权柄.md`、`精神之海.md` 与 `星辰异变.md` 负责公共机制。
- 角色的 `契约.md` 负责公共机制在该角色身上的实际落点，不重新定义权柄共通规则。
- **星辰不设星名**：设定层一律以「契主＋权柄方向」指称；个别星辰若值得有名字，让它在剧情中被命名。
- 星辰性质、成立条件、代价、登记状态与精神之海评级等细则，未拍板前不写入正式人物档案，转入施工层待确认记录承接。
- 拍板日期、更新时间、更新原因与讨论过程不写入正式人物档案，决策记录以 git 历史与施工层记录为准。

## 写作边界

- 新人物进入正典前，必须确认其身份、关系、势力、契约状态和出场位置不与既有文件冲突。
- 人物命名、正文称呼、对话昵称和视角化称呼，必须同步核对 [命名检查清单](../_命名检查清单.md)。
- 若人物设定涉及星辰契约体系、势力格局、重大关系或线索真相，先取得作者确认，再写入正式人物档案（真源顺位见 `BIBLE.md`）。
- 女主入场线随各人物卡确认。
