# 人物模板索引

> 本目录只负责人物静态模板分流，不横向展开具体人物正文或动态状态。模板存在不代表人物已经进入正典。

> `characters/` 只放静态人物档案；会随章节变化的内容请放到角色动态目录 `context/characters/`。

## 必备静态模板

新建人物时，在 `characters/major/人物名/`、`characters/minor/人物名/` 或 `characters/preentry/人物名/` 下复制：

- [主卡.md](主卡.md)
- [形象.md](形象.md)
- [衣装.md](衣装.md)

## 按需静态模板

- [星契.md](星契.md)：只在角色星契身份及至少一项权柄方向已经确认时创建；非星契者或星契状态未确认者不创建正式星契档案，缺口统一登记于 `.analysis/blueprints/待确认/人物.md`。

复制后到 `characters/_index.md` 与对应层级索引登记编号、状态和已有子档。

## 动态模板

- 动态人物状态应放入角色动态目录 `context/characters/`。
- 动态目录尚未建立统一模板前，不要把当前处境、关系推进、出场承接等动态变化塞回静态卡。

模板选用说明、星契档案创建条件和真源边界见 [README](./README.md)。
