# 亲密卡字段规范

> 本页用于统一 `intimacy/characters/` 下角色亲密卡的字段优先级、必答问题与取舍原则。角色卡是校准工具，不是玩法堆料页。
>
> 本项目口径：亲密写实＋美感修辞、不用异能滤镜（`STYLE.md`）；亲密不功利化·软支撑红线（`BIBLE.md`）——卡内不得出现增益、修为、数值类字段。

## 定位

- 亲密卡负责回答"这个角色一旦进入亲密线，最稳定的核心气味是什么"
- 亲密卡不是替代主角色卡，也不是替代场景设计卡
- 主角色卡负责"她是谁"，亲密卡负责"她在亲密里怎么成立"，设计卡负责"这一场为什么这样写"

## 必答字段

以下字段视为高优先级真源，后续补卡时应优先写稳：

- **亲密主轴**：这个角色在亲密里最核心的关系气味是什么
- **禁写提醒**：最容易把她写歪到哪里
- **语言锚点**：她怎么称呼、怎么变句式、什么时候改口、什么时候失语。卡头部的「亲密称呼」行是它的速查投影：只在称呼已确认时保留，未确认则转入施工层待确认记录。
- **主动方式**：她如何索取、引导、靠近、反客为主或递上自己
- **退让方式 / 给予方式**：她如何松口、默认、寻求安抚或完成交付
- **事后余韵**：结束后她最像她自己的状态是什么
- **越线阈值**：通常要到什么关系节点，她才会真正松口
- **羞耻来源**：她最怕被看见什么、最容易因什么乱掉
- **关系落点**：她的亲密戏最后该落到什么确认上
- **正文亲密偏好**：若正文只写到越线后的交融层，她更适合把强度落在感官、呼吸、心理、关系确认还是事后余波

## 次级字段

以下字段重要，但可以在主轴写稳后逐步细化：

- **核心反应**
- **敏感位点**
- **体质反馈**（写实体质特征）
- **情动体征**（写实的可见变化——红晕路径、体温、汗、眼神、声音）
- **路线适配**
- **多人同场动态**
- **安抚偏好**
- **场域偏好**：跨场稳定的空间倾向、职业／公共空间边界、多人空间需求与环境锚点；不记录具体场景调度
- **隐私边界**：可见与在场范围、影像接受度、拍摄观看权限、保存删除规则与身份保护要求；具体场景仍须逐次确认
- **情趣铺垫锚点**
- **关系副手感**

## 补充字段

以下字段只在真的会影响落笔时再展开，不必为了完整而强填：

- **私密构造**
- **玩具锚点**
- **深色带适配（速查）**：该角色在深色情欲带（正向线 D/s 情趣）中的档位与调用要点，五要素——档位（A/B/C）/ 入口情趣 / 内核·触发 / 禁忌 / 特异点。提炼自「玩具锚点 / 情趣引导 / 权力承受度」，不新造设定，供 skill 一眼取档；卡内放在「玩具锚点」之后。口径见 `story/genres/深色情欲带.md`；本书正向线深色带不走 C 档羞辱向，疼痛至多 B 档「调味」级。另有两个**可选要素**，均不由 A/B/C 档位推出、不写即按常规处理：**「探索广度」**——该角色愿意试的品类多寡与换新频率；广度不改变强度上限，不因「什么都试过」推出可上羞辱或明显疼痛（见口径页 §7）。**「扮演式强制」**——默认不写＝未授权；写则必须同时给出授权路径与该角色的专属约束（三项硬前置见口径页 §4）。当前全书仅苏璃授权。
- **高强度玩法逻辑**：对应模板「玩具锚点」内的适合玩法与玩法逻辑，不是独立栏位
- **多人互动癖好**
- **专属锚点**：针对特定人设的深度补强，有则填，无则删
- **亲密图谱（九阶段）**：起势到收束的分阶段语言/身体/体征锚点，只在正文需要逐阶段落笔时展开
- **HD 角色定调**：HD / B 场相关定调，只在角色进入对应场型时填写

若角色当前还没走到对应阶段，宁可留白，也不要为了"卡面完整"提前发明。

卡头部的引用行（主角色卡、形象档案、EMO代码、性格曲线）与「形象引用」章节不参与上述分档：它们的真源在主角色卡与形象档案，亲密卡只做入口引用。

## 每张卡必须回答的问题

- 她为什么会愿意
- 她怎么表示愿意
- 她怎么索取
- 她怎么收住
- 她最怕什么
- 她最吃什么
- 她被真正打动时，最像她自己的动作是什么
- 这条线和其他女主相比，最不可替代的差异是什么
- 若已进入完整亲密阶段，她的正文亲密最该抓哪一种交融感，最不该写成哪一种俗味
- 她在什么空间里最容易成立，哪些职业或公共空间不能越界
- 谁能看见、在场、拍摄、观看、保存，以及她如何撤回授权

## 写法原则

- 一张亲密卡真正稳定的核心，通常只靠 3-5 个锚点，不靠字段越多越强
- 允许局部撞词，不必为了去重把自然语言削得很怪；真正要防的是撞味、撞关系逻辑、撞收尾感觉
- 不要把同义反复分散到多个栏位里；如果"主动方式"和"关系落点"写的是同一件事，应保留最直接的一处
- 角色差异优先落在"关系逻辑、语言变化、主动/退让方式、事后余韵"，不要只落在身体细节
- 若角色已经进入完整亲密阶段，差异也应优先落在"交融方式、主客交换、失守方式、身体反馈与余波落点"，不要只用一套泛化的热、软、麻去覆盖所有人
- 如果一个角色最强的地方在关系变化，就不要用大量感官设定盖过去
- 如果一个角色常态下不会主动调情，就不要为了卡面热闹硬写会撩
- 体质、体征一律写实：不写发光、异象、能量流转；美感靠修辞与节奏，不靠异能滤镜

## 推荐补卡顺序

1. 先写 `亲密主轴`
2. 再写 `禁写提醒`
3. 再写 `语言锚点`
4. 再写 `主动方式 / 退让方式 / 事后余韵`
5. 再写 `越线阈值 / 羞耻来源 / 关系落点`
6. 再写 `场域偏好 / 隐私边界`
7. 最后再补体质反馈、路线适配、私密构造等扩展项

## 何时需要回修

- 新角色一进亲密线就和已有角色撞味
- 一场戏写完后，读不出"这为什么只能是她"
- 角色在日常暧昧、朦胧亲密和完整亲密之间像换了一个人
- 卡里写得很热闹，但落正文时抓不住一句能直接用的动作、停顿或称呼
