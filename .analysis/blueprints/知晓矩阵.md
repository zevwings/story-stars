# 知晓矩阵（多视角运行仪表盘）

> 施工层派生工具，非正式正典。多视角限知的写作纪律以本表为准；口径本身以 `STYLE.md` 视角、`BIBLE.md` 第四条为真源。

## 运行规则

- **给某角色视角＝决定把他知道的暴露给读者**。需保持神秘的幕后 / 反派不给视角，或只给「知道得不多」的下层视角承载。
- **每个关键秘密开写前对一次表**：确认它走「秘密悬念」还是「戏剧反讽」、由哪个视角承载、哪些视角段落里绝不能说破。
- **每个视角各自限知，谁都不全知**；读者靠跨视角拼图形成信息落差，可比男主先知。
- **唐家统一规则**：唐家「机构 / 机器」在阶段一一律不给视角（保持阴影）；唐家的血缘只经唐语薇本人的私人视角漏给读者（个人恐惧，不是家族权力展示）。
- **默认引擎规则**：女主的私人创伤 / 秘密默认走**悬念**、贴身男主（「陪她揭开」是亲密的一部分，反讽会削弱它）；只有身份 / 立场 / 势力类、且确有反讽价值的，才升格**反讽**。

## 常态方法（非前置审计）

- 既有 plotline 的揭开点**不做一次性大扫除**；随各线 outline 时，对着本表 ＋ 上面「默认引擎规则」逐个标即可（懒加载）。
- 新秘密出现即在本表加一行；引擎与承载视角在该线开写前定。
- 少数色带（非男主情欲）的写作纪律见 `STYLE.md` 亲密 / 高张力场景段。
